# KlinkerOblik — Технічна Специфікація

## Огляд проєкту

Telegram-бот **KlinkerOblik** для обліку робіт будівельних бригад (укладання клінкерної плитки на фасади).
Дозволяє працівникам щоденно вносити результати роботи, а керівництву — отримувати звіти.

## Ключові сутності

### Ієрархія локацій
```
Об'єкт (Project)
  └── Дім (Building)
        └── Елемент (Element): стіна, партер, тераса, балкон, стеля тощо
```

**Важливо:** Об'єкти, доми та елементи створює ТІЛЬКИ керівник або бригадир.
Працівники обирають зі списку — не мають права створювати нові.
Це запобігає дублюванню назв (коли різні бригади називають один об'єкт по-різному).

### Організаційна структура
```
Керівник фірми (Director) — верхній рівень
  └── Бригада (Brigade)
        ├── Бригадир (Brigadier) — може бути кілька на одну бригаду
        └── Працівник (Worker)
```

### Три ролі

| Роль | Код | Опис |
|------|-----|------|
| Керівник фірми | `director` | Власник акаунту. Повний доступ до всього |
| Бригадир | `brigadier` | Керує своєю бригадою. Може додавати працівників, створювати елементи |
| Працівник | `worker` | Вносить свої дані, бачить свої звіти |

### Матриця прав

| Дія | Керівник | Бригадир | Працівник |
|-----|----------|----------|-----------|
| Створити бригаду | ✅ | ❌ | ❌ |
| Призначити бригадира | ✅ | ❌ | ❌ |
| Додати працівника в бригаду | ✅ | ✅ (свою) | ❌ |
| Перемістити працівника між бригадами | ✅ | ❌ | ❌ |
| Створити об'єкт | ✅ | ✅* | ❌ |
| Створити дім / елемент | ✅ | ✅* | ❌ |
| Редагувати об'єкт/дім/елемент | ✅ | ✅* | ❌ |
| Додати тип робіт | ✅ | ❌ | ❌ |
| Змінити розцінки (замовч.) | ✅ | ❌ | ❌ |
| Змінити розцінки (по об'єкту) | ✅ | ✅* | ❌ |
| Записати роботу | ✅ | ✅ | ✅ |
| Звіти по всіх бригадах | ✅ | ❌ | ❌ |
| Звіти по своїй бригаді | ✅ | ✅ | ❌ |
| Свої особисті звіти | ✅ | ✅ | ✅ |

*✅* — бригадир може створювати об'єкти/доми/елементи, але керівник бачить єдиний список і може перейменувати/об'єднати дублі, якщо виникнуть.

### Типи робіт (WorkType)
Система повинна підтримувати довільну кількість типів робіт. За замовчуванням:

| Тип роботи | Одиниця | Ставка за замовч. |
|------------|---------|-------------------|
| Укладання плитки | м² | 60 zł |
| Погонні роботи | мп | 30 zł |
| Підрізка під 45° | мп | налаштовується |
| Прорізка делатацій | мп | налаштовується |
| Погодинна робота | год | налаштовується |
| Фугування | м² | налаштовується |
| Силікон | мп | налаштовується |

Керівник може додавати нові типи робіт з довільною назвою, одиницею та ставкою.

### Система розцінок (Rate)
- Керівник встановлює **ставки за замовчуванням** для всієї фірми
- Ставки можна **перевизначити на рівні об'єкта** (проєкту)
- Пріоритет: ставка об'єкта → ставка за замовч.
- Валюта: за замовчуванням PLN (zł), можна змінити (EUR, UAH)

## Ролі користувачів — детально

### Керівник фірми (Director)
- Створює/редагує бригади
- Призначає бригадирів
- Додає/видаляє працівників, переміщує між бригадами
- Створює/редагує об'єкти, доми, елементи
- Додає/редагує типи робіт
- Встановлює ставки (за замовч. та за об'єктом)
- Отримує звіти: день/тиждень/місяць, по всіх бригадах/об'єктах/елементах
- Бачить дані ВСІХ бригад

### Бригадир (Brigadier)
- Додає працівників у свою бригаду (через код запрошення)
- Може створювати об'єкти, доми, елементи
- Може встановлювати розцінки по об'єкту
- Вносить свої результати роботи
- Бачить звіти по своїй бригаді
- Може бути кілька бригадирів на одну бригаду

### Працівник (Worker)
- Вносить результати роботи за день
- Бачить свій заробіток за день/тиждень/місяць
- Обирає об'єкт → дім → елемент зі списку (не створює нові!)
- Може змінювати дату запису (для коригування)

## Функціональність бота

### Реєстрація / Перший запуск

**Новий користувач:**
1. `/start` → Вибір мови (🇵🇱 Polski / 🇺🇦 Українська / 🇷🇺 Русский)
2. Три шляхи:
   - **"Я керівник фірми"** → Створює акаунт директора → Створює першу бригаду → Призначає бригадира
   - **"Я бригадир"** → Вводить код запрошення від директора → Отримує роль бригадира
   - **"Я працівник"** → Вводить код запрошення від бригадира або директора → Приєднується до бригади

**Реєстрація директора:**
- Ім'я
- Телефон (автоматично з Telegram)
- Email (опціонально)
- Назва фірми/компанії (опціонально)

**Приєднання бригадира:**
- Ім'я
- Код запрошення (з роллю `brigadier`)
- Автоматична прив'язка до бригади з правами бригадира

**Приєднання працівника:**
- Ім'я
- Код запрошення (з роллю `worker`)
- Автоматична прив'язка до бригади

**Коди запрошення:**
- Генеруються директором або бригадиром
- Містять інформацію про роль (brigadier / worker) і бригаду
- Одноразові, з терміном дії (7 днів за замовч.)
- Формат: 6-символьний код (наприклад: `KL-A7X9`)

---

### Меню працівника

```
📝 Записати роботу
📊 Мої звіти
⚙️ Налаштування (мова, валюта)
```

#### 📝 Записати роботу
1. Вибір дати (за замовч. — сьогодні, кнопка "Інша дата")
2. Вибір об'єкта → Inline-кнопки зі списком (тільки вибір, не створення!)
3. Вибір дому → Inline-кнопки
4. Вибір елемента → Inline-кнопки
5. Послідовний ввід об'ємів:
   - "Введіть м² (укладання плитки):" → користувач пише число (або 0 / пропустити)
   - "Введіть мп (погонні):" → число
   - Далі по кожному активному типу робіт
6. Підсумок:
   ```
   ✅ Записано: 22.03.2026
   📍 Osiedle Parkowe → Dom 3 → Ściana 2

   Укладання плитки: 12.5 м² × 60 zł = 750.00 zł
   Погонні: 8 мп × 30 zł = 240.00 zł
   Підрізка 45°: 3 мп × 25 zł = 75.00 zł

   💰 Разом: 1 065.00 zł
   ```
7. Кнопки: [✅ Підтвердити] [✏️ Змінити] [❌ Скасувати]

#### 📊 Мої звіти
- За сьогодні
- За тиждень (поточний)
- За місяць (поточний)
- За довільний період
- По об'єкту (вибір)

---

### Меню адміна (Директора)

```
📝 Записати роботу
📊 Звіти
👷 Бригади та працівники
🏗️ Об'єкти
💰 Розцінки
🔧 Типи робіт
⚙️ Налаштування
```

#### 👷 Бригади та працівники
- Список бригад → в кожній: бригадири та працівники
- Створити нову бригаду
- Призначити бригадира (з існуючих працівників або новий код запрошення з роллю)
- Створити код запрошення (вказати роль: бригадир / працівник)
- Перемістити працівника між бригадами
- Деактивувати працівника (не видаляти)

### Меню бригадира

```
📝 Записати роботу
📊 Звіти (своя бригада)
👷 Працівники (своя бригада)
🏗️ Об'єкти
⚙️ Налаштування
```

#### 👷 Працівники (бригадир)
- Список працівників своєї бригади
- Створити код запрошення для працівника
- Деактивувати працівника

#### 🏗️ Об'єкти
- Список об'єктів
- Створити об'єкт → Додати доми → Додати елементи
- Редагувати (перейменувати, додати дім/елемент)
- Архівувати завершений об'єкт

#### 💰 Розцінки
- Переглянути ставки за замовч.
- Змінити ставку за замовч.
- Встановити ставки для конкретного об'єкта
- Вибір валюти (PLN / EUR / UAH)

#### 🔧 Типи робіт
- Список активних типів
- Додати новий тип (назва, одиниця виміру, ставка за замовч.)
- Деактивувати тип (не видаляти — щоб не зламати старі записи)

#### 📊 Звіти (адмін)
Фільтри:
- Період: день / тиждень / місяць / довільний
- Бригада: всі / конкретна
- Працівник: всі / конкретний
- Об'єкт: всі / конкретний
- Дім: всі / конкретний
- Елемент: всі / конкретний

Формат звіту:
```
📊 Звіт за 18.03 - 22.03.2026
🏗️ Об'єкт: Osiedle Parkowe

👷 Бригада "Альфа":
  Іван Петренко:
    Dom 1 / Ściana 1: 45 м², 12 мп
    Dom 1 / Ściana 2: 32 м², 8 мп
  Олег Коваль:
    Dom 2 / Parter: 28 м², 15 мп

Разом бригада "Альфа":
  м²: 105 × 60 zł = 6 300 zł
  мп: 35 × 30 zł = 1 050 zł
  💰 Всього: 7 350 zł

📋 ЗАГАЛЬНИЙ ПІДСУМОК:
  м²: 210
  мп: 67
  💰 Всього: 14 700 zł
```

Можливість експорту в Excel (CSV).

## Технічний стек

- **Мова:** Python 3.11+
- **Фреймворк бота:** python-telegram-bot (v20+, async)
- **База даних:** PostgreSQL (через asyncpg або SQLAlchemy async)
- **ORM:** SQLAlchemy 2.0 (async)
- **Міграції:** Alembic
- **Сервер:** Hetzner CX22 (2 vCPU, 4 GB RAM, €3.85/міс)
- **ОС:** Ubuntu 24.04 LTS
- **Деплой:** Docker Compose (бот + PostgreSQL)
- **Бекапи:** автоматичні щоденні (pg_dump → зберігання)

---

## Мультитенантність (вбудована з самого початку)

- Кожен директор = окремий "тенант"
- Повна ізоляція даних між тенантами через `director_id`
- Директор бачить тільки свої бригади/об'єкти/записи
- Бригадир бачить тільки свою бригаду
- Працівник бачить тільки свої записи

---

## Безпека

- Дані зберігаються в ЄС (Hetzner, Німеччина)
- Шифрування з'єднання (SSL/TLS)
- Авторизація через Telegram ID (без паролів)
- Коди запрошення — одноразові, з терміном дії
- Логування дій директора та бригадирів
- Регулярні бекапи

---

## Мовні коди

Всі тексти інтерфейсу зберігаються в окремих файлах локалізації:
- `locales/uk.json` — Українська
- `locales/pl.json` — Polski
- `locales/ru.json` — Русский

Додавання нових мов — просто додати новий файл.
